Preliminary Features

Certain sections of this documentation may be marked as preliminary documentation. This page describes the role of classes and methods marked as preliminary in a release of this library.

Adding Features

New features SHOULD be added with the preliminary marking.

  • Breaking changes to features marked as preliminary are allowed in releases that do not change the assembly's strong name, provided they are listed in the release notes.

  • If any binary breaking changes were made to a feature while it was marked preliminary, the assembly strong name MUST be updated when the preliminary marking is removed. If no binary breaking changes were made to a feature while it was marked preliminary, the preliminary marking MAY be removed without changing the strong name of the assembly.

Breaking Changes

  • Breaking changes to binary compatibility MUST NOT be made to features without updating the assembly's strong name unless they are marked preliminary.

  • Breaking changes to source compatibility SHOULD NOT be made to features without updating the assembly's strong name, unless they are marked preliminary.
