                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                       Notes to Callers

                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                          
             If the resource function throws an exception, or if the Task TResult  it returns
             does not complete successfully, the resource will not be acquired by this method. In either of these
             situations the caller is responsible for ensuring the resource function cleans up any
             resources it creates.
